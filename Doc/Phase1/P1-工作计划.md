## P1-5：数据库迁移【最先执行，后端】

> 调整自 Review 意见 2：提前到所有功能开发之前。

| 任务 | 说明 |
|------|------|
| 确认 PostgreSQL 已安装 | 版本 ≥ 14 |
| 复制 `.env.example` → `.env` | 填写真实 `DATABASE_URL` |
| `npm run db:migrate` | 生成 migration 文件并建表 |
| `npm run db:generate` | 生成 Prisma Client |
| 验证现有路由 | 将 `USE_MOCK=false`，启动后端，调用 `/api/health`、`/api/inventory`、`/api/orders`，确认 Prisma Client 无报错 |
| 恢复 Mock 模式 | 验证完成后可切回 `USE_MOCK=true` 继续开发 |

## P1-2：库存管理页【前后端联调】

**目标：** 完整的 FBA 库存列表，支持搜索、过滤、低库存预警高亮。

### 前端任务

| 任务 | 说明 | 优先级 |
|------|------|--------|
| Ant Design Table | 列：SKU / ASIN / 商品名 / 可售量 / 在途量 / 预警阈值 | P0 |
| 低库存行高亮 | `fbaAvailable <= reorderThreshold` 时行背景标红 | P0 |
| 搜索框 | 按 SKU / ASIN / 商品名模糊搜索（`getInventory({ search })` ） | P0 |
| 低库存开关 | "仅显示低库存" Switch（`getInventory({ lowStockOnly: true })` ） | P1 |
| 商品详情侧边栏 | 点击行展开 Drawer，显示 ASIN / 主图 / 补货参数 | P1 |

### 后端验证任务

| 任务 | 说明 |
|------|------|
| 验证 `GET /api/inventory?search=xxx` | 模糊匹配正常 |

| 验证 `GET /api/inventory/:asin` | 返回单品详情正常 |

## P1-3：订单管理页【前后端联调】

**目标：** 订单列表 + 状态筛选 + 详情查看，不含任何买家 PII。

### 关于物流追踪号字段

> 调整自 Review 意见 3：`Order` 表和 `OrderItem` 表目前**没有 `trackingNumber` 字段**。

**决定：** 本 Phase 订单详情 Drawer **暂不展示物流追踪号**，等 SP-API 真实对接时补充 Schema 字段 + migration，避免现在造无意义 Mock 数据。

### 前端任务

| 任务 | 说明 | 优先级 |
|------|------|--------|
| Ant Design Table | 列：订单号 / 状态 Tag / 金额 / 下单时间 / 最后更新时间 | P0 |
| 状态筛选 Select | 支持 6 种 OrderStatus | P0 |
| 日期范围 DatePicker | 按下单日期过滤 | P1 |
| 分页 | 由后端返回 `pagination` 字段驱动 | P0 |
| 订单详情 Drawer | 展示商品明细（SKU / 数量 / 单价），**不展示买家信息，不展示物流追踪号（待 SP-API 对接后补）** | P1 |
| 异常订单高亮 | CANCELED / UNFULFILLABLE 状态标红 Tag | P1 |

### 后端验证任务

| 任务 | 说明 |
|------|------|
| 验证 `GET /api/orders?status=SHIPPED&page=1&pageSize=20` | 状态过滤 + 分页正常 |
| 验证 `GET /api/orders/:amazonOrderId` | 返回含 OrderItem 的订单详情正常 |
