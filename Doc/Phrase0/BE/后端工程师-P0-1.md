# Role
You are a senior backend engineer building REST API routes with mock data support.

# Context
Project: Amazon Seller Dashboard (monorepo)
- server/ contains Express + Prisma setup (already scaffolded by another agent)
- USE_MOCK=true in .env means return mock data instead of calling SP-API
- All prices use Decimal(10,2), no buyer PII stored anywhere

# Task: Backend Routes + Mock Data Layer

## 1. Create server/src/mock/data.js

Provide realistic mock data for a US FBA seller. Include:

Products (6 items, mix of single ASINs and 2 parent/variant groups):
- Vary fbaAvailable: some healthy (>50), some low (<30, below reorderThreshold), one at 0
- All fields populated: asin, sku, parentAsin (null or filled), title, imageUrl (use
  placeholder "https://via.placeholder.com/100"), price, fbaAvailable, fbaInbound,
  reorderThreshold, leadTimeDays, safetyBufferDays, salesWindowDays

Orders (10 items, spread across OrderStatus values):
- Mix of PENDING, UNSHIPPED, SHIPPED, DELIVERED, CANCELED
- purchaseDate spread across last 30 days
- Each order has 1-2 OrderItems linked to the mock products above
- No buyer information of any kind

SyncLogs (5 entries):
- Mix of SUCCESS, WARNING, FAILURE across INVENTORY and ORDERS modules
- Realistic errorMessage for WARNING and FAILURE entries
- Realistic durationMs (300-2000ms range)

## 2. Create server/src/services/mockService.js

Export functions that mirror what the real SP-API service will provide:
- getInventory() → returns mock products array
- getOrders(filters) → accepts { status, dateFrom, dateTo } filters, returns filtered orders
- getOrderById(amazonOrderId) → returns single order with items
- getDashboardStats() → returns:
  {
    totalSkus: number,
    lowStockCount: number,       // fbaAvailable <= reorderThreshold
    todayOrders: number,
    todayRevenue: Decimal,
    monthOrders: number,
    monthRevenue: Decimal,
    lastSyncAt: ISO string
  }

## 5. Create server/src/routes/inventory.js

GET /api/inventory
- Query params: search (string), lowStockOnly (boolean)
- Filter mock data accordingly
- Response: { data: Product[], total: number, lastSyncAt: string }

GET /api/inventory/:asin
- Returns single product or 404

## 6. Create server/src/routes/orders.js

GET /api/orders
- Query params: status, dateFrom, dateTo, page (default 1), pageSize (default 20)
- Response: { data: Order[], total: number, page: number, pageSize: number }

GET /api/orders/:amazonOrderId
- Returns order with items or 404

## 7. Create server/src/routes/dashboard.js

GET /api/dashboard/stats
- Returns getDashboardStats() result

- In mock mode: simulate a 500ms delay, return success SyncLog entry
- Response: { success: true, log: SyncLog }
