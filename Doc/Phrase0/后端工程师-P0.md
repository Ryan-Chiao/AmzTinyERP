# 角色

你是一名资深后端工程师，负责搭建一个新项目的基础脚手架。

# 项目背景

Amazon Seller Dashboard —— 一个面向美国亚马逊 FBA 卖家的自托管运营工具，单账号自用。
技术栈：Node.js + Express（后端）、React + Vite + Ant Design（前端）、PostgreSQL + Prisma ORM（数据库）。采用 Monorepo 结构。

## 2. 将以下 Prisma Schema 写入 server/prisma/schema.prisma

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model Product {
  id                 String   @id @default(cuid())
  asin               String   @unique
  sku                String   @unique
  parentAsin         String?
  title              String
  imageUrl           String?
  price              Decimal  @db.Decimal(10, 2)
  fbaAvailable       Int      @default(0)
  fbaInbound         Int      @default(0)
  reorderThreshold   Int      @default(30)
  leadTimeDays       Int      @default(30)
  safetyBufferDays   Int      @default(7)
  salesWindowDays    Int      @default(30)
  createdAt          DateTime @default(now())
  updatedAt          DateTime @updatedAt
  orderItems         OrderItem[]
  inventorySnapshots InventorySnapshot[]
  @@index([parentAsin])
}

model InventorySnapshot {
  id           String      @id @default(cuid())
  productId    String
  product      Product     @relation(fields: [productId], references: [id])
  fbaAvailable Int
  fbaInbound   Int
  triggerType  SyncTrigger
  recordedAt   DateTime    @default(now())
  @@index([productId, recordedAt])
}

model Order {
  id              String      @id @default(cuid())
  amazonOrderId   String      @unique
  status          OrderStatus
  totalAmount     Decimal     @db.Decimal(10, 2)
  currency        String      @default("USD")
  purchaseDate    DateTime
  lastUpdateDate  DateTime
  createdAt       DateTime    @default(now())
  updatedAt       DateTime    @updatedAt
  orderItems      OrderItem[]
}

enum OrderStatus {
  PENDING
  UNSHIPPED
  SHIPPED
  DELIVERED
  CANCELED
  UNFULFILLABLE
}

model OrderItem {
  id        String   @id @default(cuid())
  orderId   String
  order     Order    @relation(fields: [orderId], references: [id])
  productId String?
  product   Product? @relation(fields: [productId], references: [id])
  asin      String
  sku       String?
  title     String
  quantity  Int
  price     Decimal  @db.Decimal(10, 2)
  @@index([orderId])
  @@index([asin])
}

model SyncLog {
  id           String      @id @default(cuid())
  module       SyncModule
  triggerType  SyncTrigger
  status       SyncStatus
  errorMessage String?
  recordCount  Int?
  durationMs   Int?
  createdAt    DateTime    @default(now())
  @@index([module, createdAt])
  @@index([status, createdAt])
}

enum SyncModule {
  INVENTORY
  ORDERS
  PRICING
  ADVERTISING
}

enum SyncTrigger {
  SCHEDULED
  MANUAL
  STARTUP
}

enum SyncStatus {
  SUCCESS
  WARNING
  FAILURE
}
```

## 3. 创建 server/src/index.js（最小可运行 Express 服务）

要求：
- 加载 dotenv
- 挂载 CORS 中间件（允许所有来源，仅本地使用）
- 提供 `GET /api/health` 接口，返回 `{ status: "ok", timestamp: new Date().toISOString() }`
- 从 `.env` 读取 PORT，默认 3001
- 启动后在控制台打印：`服务已启动，监听端口 PORT`

## 6. 创建 server/package.json

依赖（dependencies）：express、@prisma/client、prisma、dotenv、cors、node-cron

开发依赖（devDependencies）：nodemon

Scripts：
- `"dev"`: `"nodemon src/index.js"`
- `"db:migrate"`: `"prisma migrate dev"`
- `"db:generate"`: `"prisma generate"`

## 8. 创建 README.md

包含以下内容：

**前置条件：**
- Node.js 18+
- PostgreSQL 14+

**启动步骤：**
1. `cd server && npm install`
2. 复制 `.env.example` 为 `.env`，填写 `DATABASE_URL`
3. `npm run db:migrate`
4. `npm run dev`

**注意事项：**
- 设置 `USE_MOCK=true` 可在没有 SP-API 凭证的情况下运行
- Windows 用户请使用 PowerShell 或 Git Bash 执行 npm 脚本
