## 关键设计决策

- **Vite Proxy**：前端所有 `/api/*` 请求经 Vite devServer 代理转发，避免 CORS，生产部署时改 nginx 即可
- **npm 代理陷阱**：环境变量 `HTTP_PROXY=127.0.0.1:10808` 导致 npm 假死，安装时须临时清空；建议在不使用代理时执行 `npm config delete proxy && npm config delete https-proxy`
- **Error Boundary 层级**：Dashboard 页内自行处理 loading / error state，不依赖全局 ErrorBoundary，便于局部 retry
- **路由懒加载**：当前为直接 import，页面增多后可改用 `React.lazy` + `Suspense`
