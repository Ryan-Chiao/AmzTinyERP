## 1. 项目概述

### 1.1 背景与目标

本工具面向美国亚马逊 FBA 卖家自用，通过对接 Amazon SP-API 实现库存、订单、定价及广告数据的统一管理，替代手动登录 Seller Central 查看数据的低效方式。

**核心目标：**

- 减少信息分散，一个界面看清运营全貌
- 库存预警，减少 FBA 断货损失
- 订单追踪，快速定位异常
- 定价与广告数据可视化，辅助决策

### 1.2 产品范围

| 维度        | 内容                         |
| --------- | -------------------------- |
| 目标用户      | 自用（单一卖家账号）                 |
| 目标市场      | Amazon 美国站（US Marketplace） |
| 发货模式      | FBA 为主，兼容 FBM              |
| 访问方式      | 本地 Web 应用，浏览器访问            |
| SP-API 状态 | 注册进行中                      |

### 1.3 不在范围内（Out of Scope）

- 多店铺管理（Phase 3 再议）
- 对外 SaaS 化（Phase 3 再议）
- 自动下单补货（只做建议，不自动执行）
- 自动改价（只做建议，不自动执行）
- **买家 PII 数据**（姓名、邮件、电话、收货地址）— 不采集、不存储、不展示

  ├── 订单管理页
  ├── 定价/补货页（Phase 2）
  └── 广告数据页（Phase 2）

### 3.2 Phase 1 — 库存管理

**目标：** 实时掌握 FBA 库存状态，提前预警断货风险。

#### 功能列表

| 功能 | 优先级 | 说明 |
|------|--------|------|
| FBA 库存列表 | P0 | 展示所有 SKU 的可售数量、在途数量 |
| 低库存预警 | P0 | 可自定义阈值，低于阈值标红提示 |
| 手动刷新 | P0 | 不等定时任务，手动触发同步 |
| 库存历史趋势 | P1 | 折线图展示近 30/60/90 天库存变化 |
| 补货建议 | P1 | 基于销售速度估算建议补货时间和数量 |
| 商品基础信息 | P1 | ASIN、SKU、标题、主图、价格 |

#### 补货建议算法

```
日均销量 = 近 30 天销量 / 30
建议补货数量 = 日均销量 × (补货周期天数 + 安全缓冲天数) - 当前可售库存
预计断货日期 = 今天 + (当前可售库存 / 日均销量)

参数说明：
  - 补货周期天数：用户手动设置（头程约 30-45 天）
  - 安全缓冲天数：用户手动设置（建议 7-14 天）
```

#### 数据字段

```
Product 表
- id, asin, sku, title, image_url
- price, fba_available, fba_inbound
- reorder_threshold（自定义预警值）
- restock_lead_days（补货周期，用户设置）
- safety_buffer_days（安全缓冲，用户设置）
- created_at, updated_at
```

#### 预警逻辑

```
当 fba_available ≤ reorder_threshold 时：
- 列表行标红
- Dashboard 显示预警数量（可点击跳转）
- （Phase 2）支持邮件/Webhook 通知
```

## 4. 数据同步策略

### 4.1 同步方式

| 方式 | 频率 | 说明 |
|------|------|------|
| 定时增量同步 | 每小时 | Orders 按 `LastUpdateDate` 增量拉取，减少 API 配额消耗 |
| 全量同步 | 每天凌晨 | 库存全量刷新，保证数据一致性 |
| 手动触发 | 用户点击刷新 | 即时同步，前端显示 loading 状态 |

### 4.2 Rate Limit 处理

```
策略：指数退避重试
- 收到 429 响应 → 等待 2^n 秒后重试（最多 3 次）
- 请求队列：同一时间最多 2 个并发 SP-API 请求
- 每次同步前检查配额，超限则跳过本次
```

### 4.3 SyncLog 表

```
SyncLog 表
- id
- module（inventory / orders / pricing / ads）
- synced_at（同步完成时间）
- status（success / error）
- records_count（本次同步条数）
- error_message（失败原因，成功时为空）
```

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 数据同步频率 | 增量每小时，全量每日，支持手动触发 |
| 响应速度 | 页面加载 < 2秒（本地数据库查询） |
| 安全性 | SP-API Token 存储于 `.env`，不提交 git |
| PII 合规 | 不申请、不采集、不存储任何买家 PII 数据 |
| 数据保留 | 本地 PostgreSQL，保留 12 个月；超期数据每月自动清理 |
| 错误处理 | 分级处理（见下表） |

### 错误处理分级

| 错误类型 | 处理方式 |
|----------|----------|
| Token 过期（401） | 自动静默刷新 Token，用户无感知 |
| Rate Limit（429） | 指数退避重试，前端显示"同步中"状态 |
| 网络超时 | 显示缓存数据 + Toast 提示"使用本地缓存" |
| API 返回错误 | 显示上次同步时间，不白屏，记录 SyncLog |
